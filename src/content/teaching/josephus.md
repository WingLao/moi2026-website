# Josephus 整理

## 1. 主題總覽

| 項目 | 內容 |
| --- | --- |
| 題目模型 | `n` 個人圍成一圈，每數到第 `k` 個人就出圈 |
| 常見要求 | 求完整出圈順序、求最後存活者 |
| 核心能力 | 環狀思維、取模 `%`、模擬、遞推 |
| 做題先判斷 | 題目要整個過程，還是只要最後答案 |

## 2. 兩種解法

| 解法 | 適合情況 | 核心想法 |
| --- | --- | --- |
| `list` 模擬 | 題目要完整出圈順序 | 每次算出要刪除的位置，再把那人移除 |
| 遞推 | 題目只求最後存活者 | 由 `n-1` 人的答案推出 `n` 人的答案 |

## 3. 核心公式

### 3.1 模擬版

```python
idx = (idx + k - 1) % len(people)
```

### 3.2 遞推版

```python
f(1, k) = 0
f(n, k) = (f(n - 1, k) + k) % n
```

如果題目編號是 `1 ~ n`，最後要：

```python
f(n, k) + 1
```

## 4. 基本範例

![Josephus 例題：n = 7, k = 3 時，沿著圓圈每數到第 3 個人就淘汰。](/teaching/josephus-circle.svg)

| 題目 | 結果 |
| --- | --- |
| `n = 7, k = 3` | 出圈順序：`[3, 6, 2, 7, 5, 1, 4]` |
| `n = 7, k = 3` | 最後存活者：`4` |
| `n = 5, k = 2` | 最後存活者：`3` |

## 5. Python 參考解

### 5.1 輸出完整出圈順序

```python
def josephus_order(n, k):
    people = list(range(1, n + 1))
    order = []
    idx = 0

    while people:
        idx = (idx + k - 1) % len(people)
        order.append(people.pop(idx))

    return order
```

### 5.2 只求最後存活者

```python
def josephus_last(n, k):
    ans = 0
    for i in range(2, n + 1):
        ans = (ans + k) % i
    return ans + 1
```

## 6. 例題整理

| 難度 | 題目 | 建議方法 | 重點 |
| --- | --- | --- | --- |
| 易 | `n = 5, k = 2`，求最後存活者 | 模擬或遞推 | 建立直覺 |
| 中 | `n = 7, k = 3`，求完整出圈順序 | 模擬 | 練 `%` 和 `k - 1` |
| 難 | `n = 10^5, k = 7`，只求最後存活者 | 遞推 | 避免大資料模擬 |

## 7. 常見錯誤

| 錯誤 | 說明 | 正確觀念 |
| --- | --- | --- |
| 忘記 `k - 1` | 會把刪除位置算錯 | 刪除位置應從當前位置往後數 `k - 1` |
| 0-based / 1-based 混亂 | 公式結果直接輸出會錯 | 遞推通常先算 0-based，最後視題意 `+1` |
| 大資料仍硬做模擬 | `pop(index)` 反覆搬動元素，較慢 | 只求最後答案時優先用遞推 |
