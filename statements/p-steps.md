# 迷宮最短路 (steps)

> 此網站版本改用 Markdown 題面，範例輸入 / 輸出直接對齊 `data` 內的 `*-0.in` / `*-0.out`，避免題面與測資不一致。

|項目|內容|
|---|---|
|任務名稱|`steps`|
|組別|`P`|

|原始 PDF|`P-steps.pdf`|

## 題目說明

給定一個 N×M 的網格迷宮，"0" 表示可以通行，"1" 表示牆壁。從左上角 (0,0)
出發，每步可以向上下左右移動一格，求到達右下角 (N-1, M-1) 的最少步數。若無法到達，輸出 -1。

## 輸入格式

第一行包含兩個正整數 N、M（2 ≤ N, M ≤ 10）。
接下來 N 行，每行 M 個字元，為 "0" 或 "1"。

## 輸出格式

一行，輸出最少步數，若無法到達則輸出 -1。

## 範例輸入

```text
3 3
000
010
000
```

## 範例輸出

```text
4
```

## 說明

路徑為 (0,0)→(0,1)→(0,2)→(1,2)→(2,2)，共4步。
